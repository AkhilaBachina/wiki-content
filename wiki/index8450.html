[edit] Copying testcases from old release repository to current release repository

This task of copying testcases and all relevant files from an old release's testcase repository to the current release's repository can be done in several different ways.

[edit] Copying from repository to checked out working directory of new release's repository

  • source repository in the examples: rel_4.charm
  • target repository in the examples: rel_5.cat

[edit] Check out the target repository so that you have a working copy 

ssh $USER@flexiaf1.fp.nsn-rdnet.net
mkdir $HOME/testcases
cd $HOME/testcases
svn co http://sugar.fp.nsn-rdnet.net/svn/testcases/rel_5.cat rel_5.cat

(further instructions can be found here)

[edit] Locate the testcase you want to copy in the old release's repository

locate the old testcase from here: http://sugar.fp.nsn-rdnet.net/svn/testcases/rel_4.charm/FunctionalTesting/

[edit] Copy the testcase from the source repository to the working copy

Note: the working copy was created in the first step 

svn copy http://sugar.fp.nsn-rdnet.net/svn/testcases/rel_4.charm/FunctionalTesting/<domain>/<subdomain>/<testcase> $HOME/testcases/rel_5.cat/FunctionalTesting/<newdomain>/<newsubdomain>/<testcase>

Example: Copy the testcase HAS/SS_CFM/FT_HAS_CFM_027. 

 svn copy http://sugar.fp.nsn-rdnet.net/svn/testcases/rel_4.charm/FunctionalTesting/HighAvailabilityServices/SS_CFM/FT_HAS_CFM_027 $HOME/testcases/rel_5.cat/FunctionalTesting/HighAvailabilityServices/SS_CFM/

[edit] Copy common files

[edit] Copy files from <domain>/common/
  1. check which common files are in the <domain>/common directory
  2. copy those files which are still needed by the copied testcase 
svn copy http://sugar.fp.nsn-rdnet.net/svn/testcases/rel_4.charm/FunctionalTesting/<domain>/common/<file> $HOME/testcases/rel_5.cat/FunctionalTesting/<domain>/common/
[edit] Copy files from <domain>/<subdomain>/common/
  1. check which common files are in the <domain>/<subdomain>/common directory
  2. copy those files which are still needed by the copied testcase 
svn copy http://sugar.fp.nsn-rdnet.net/svn/testcases/rel_4.charm/FunctionalTesting/<domain>/<subdomain>/common/<file> $HOME/testcases/rel_5.cat/FunctionalTesting/<domain>/<subdomain>/common/

[edit] Copy domain.xml files

[edit] Copy <domain>/domain.xml

check if the source's <domain>/domain.xml contains anything

  1. if yes, check if the target's <domain>/domain.xml contains already something
    1. if no, overwrite the empty <domain>/domain.xml file as follows:
      svn cat http://sugar.fp.nsn-rdnet.net/svn/testcases/rel_4.charm/FunctionalTesting/<domain>/domain.xml > $HOME/testcases/rel_5.cat/FunctionalTesting/<domain>/domain.xml
      NOTE: This is a different svn command than before!
    2. if yes, compare the two files manually and merge the content as needed
  2. if no, no action needed.
[edit] Copy <domain>/<subdomain>/domain.xml

check if the source's <domain>/<subdomain>/domain.xml contains anything

  1. if yes, check if the target's <domain>/<subdomain>/domain.xml contains already something
    1. if no, overwrite the empty <domain>/<subdomain>/domain.xml file as follows:
      svn cat http://sugar.fp.nsn-rdnet.net/svn/testcases/rel_4.charm/FunctionalTesting/<domain>/<subdomain>/domain.xml > $HOME/testcases/rel_5.cat/FunctionalTesting/<domain>/<subdomain>/domain.xml
      NOTE: This is again not svn the svn copy command!
    2. if yes, compare the two files manually and merge the content as needed
  2. if no, no action needed.

[edit] Try-run the copied testcase

  1. Run the copied testcase on a FP5-Cat target with the --local option
    NOTE: You have to set up the FlexiTAF config file correctly (for instructions see for example here)
    afcli -t <target> -r 5.cat --local exec <testcase>
  2. check the execution results that have been stored locally (particularly the following files: tc_out, tc_analysis, tcexec_out/tcexec_out_tc)
  3. repeat step 1 and 2 until the testcase is correct and passing

[edit] Make the testcase visible for everybody

Commit the copied testcase and the potential changes to the repository. 

svn commit -m "<your message here>" <testcase>


[edit] Copying using two checked out working copies

  • source in the examples: working directory of rel_4.charm
  • target in the examples: working directory of rel_5.cat

No further details provided because this is a little bit error-prone -- one can easily copy local changes from the source working directory....
