FlexiTAF build task migration from ClearCase to SVN environment

FlexiTAF testcase build task has now support for making test application compilations in LinSEE SVN environment. These instructions describe all the necessary changes that are needed in testcase metadata files and in makefile / build.xml.

Metadata changes

Metadata build task "provider" attribute value needs to be changed from "linsee" to "linsee-svn".

C/C++ compilation needs some changes to "make-*" attributes. Compilation in SVN environment needs also to know the subsystem dependencies. Those are given with "subsystems" parameter as can be seen from the example below.

Original build task: 

<build name="FT_AS_CppAlarm_001test application"
       provider="linsee"
       make-prog="clearmake"
       make-args="-C gnu"
       makefile-path="../common/FT_AS_CppAlarm_001/makefile">
    <input>
        <fileset path="../common/FT_AS_CppAlarm_001">
            <glob pattern="*.cpp"/>
        </fileset>
        <file path="../common/FT_AS_CppAlarm_001/makefile"/>
    </input>
    <output>
        <file path="../common/FT_AS_CppAlarm_001/cpptest"/>
    </output>
</build>

New build task: 

<build name="FT_AS_CppAlarm_001test application"
       provider="linsee-svn"
       make-prog="make"
       make-args=""
       makefile-path="../common/FT_AS_CppAlarm_001/makefile">
    <parameters>
        <parameter name="subsystems" value="ss_cppalarm"/>
    </parameters>
    <input>
        <fileset path="../common/FT_AS_CppAlarm_001">
            <glob pattern="*.cpp"/>
        </fileset>
        <file path="../common/FT_AS_CppAlarm_001/makefile"/>
    </input>
    <output>
        <file path="../common/FT_AS_CppAlarm_001/cpptest"/>
    </output>
</build>

Multiple subsystem dependencies are given as a comma separated list, e.g. "ss_subsys1, ss_subsys2".

There is also a new "host" build attribute. This can be used in multi architecture builds to indicate target host where the HW architecture for the build environment is resolved, e.g. host="as-0".

You can also remove attribute "make-args" instead of leaving it empty.

Makefile and build.xml changes

In makefiles you need to make only one change:

  • include /vobs/build/path.mk -> $(VOBTAG)/build/path.mk

In build.xml files there might be more to change. Example below.

Original build.xml: 

<!DOCTYPE project [
 <!ENTITY family.properties SYSTEM "file:/vobs/build/ant/family.properties">
 <!ENTITY product.properties SYSTEM "file:/vobs/build/ant/product.properties">
 <!ENTITY common.properties SYSTEM "file:/vobs/build/ant/common.properties">
 <!ENTITY subtarget.properties SYSTEM "file:/vobs/build/ant/subtarget.properties">
 <!ENTITY rpmstuff.properties SYSTEM "file:/vobs/build/ant/rpmstuff.properties">
]>
<project name="AlarmManagerTest" default="dist" basedir=".">
  &family.properties;
  &product.properties;
  &common.properties;
  &subtarget.properties;
  &rpmstuff.properties;
  ... file continues ...

New build.xml: 

<project name="AlarmManagerTest" default="dist" basedir=".">
  <property environment="env"/>
  <import file="${env.VOBTAG}/flexiserver/build/ant/family.properties.project"/>
  <import file="${env.VOBTAG}/flexiserver/build/ant/product.properties.project"/>
  <import file="${env.VOBTAG}/flexiserver/build/ant/common.properties.project"/>
  <import file="${env.VOBTAG}/flexiserver/build/ant/subtarget.properties.project"/>
  <import file="${env.VOBTAG}/flexiserver/build/ant/rpmstuff.properties.project"/>
  ... file continues ...

Resolving subsystem dependencies

In case of build.xml files you can see subsystems directly from DIR_SS_* properties.

In case of makefiles you need to check used DIR_SS_* variables and names of libraries and interfaces. Below are some names commonly used in library and include macros and the corresponding subsystems.

  • libalarmagent.so -> ss_alarmagent
  • libaputils.so -> ss_libnmlm
  • libcalarm.so -> ss_calarm
  • libcommutil.so -> ss_commutil
  • libcppalarm.so -> ss_cppalarm
  • libcpplogapi.so -> ss_cpplog
  • libcppprefs.so -> ss_cppprefs
  • libcredServCpp.so -> ss_libcredserv
  • libcredServ.so -> ss_libcredserv
  • libdbclass.so -> ss_dbint
  • libDiameter.so -> ss_dia
  • libficm-cpp.so -> ss_ficm
  • libficm-c.so -> ss_ficm
  • libgetbindipcpp.so -> ss_libgetbindip
  • libgetbindip.so -> ss_libgetbindip
  • libhaslib.so -> ss_pro
  • libpm9agent.so -> ss_pm9
  • libpmgmt.so -> ss_libpmgmt
  • OPEN_LDAP_HOME -> ss_ldap
  • XERCES_HOME -> ss_xercesc
  • DIR_I_AAA -> ss_libcredserv
  • DIR_I_ALARMAGENT -> ss_alarmagent
  • DIR_I_ALARMLIGHTLIB -> ss_alarmlightlib
  • DIR_I_CALARM -> ss_calarm
  • I_COMMUTIL -> ss_commutil
  • I_CONFMGMT -> ss_confmgmt
  • I_CPPALARM -> ss_cppalarm
  • I_CPPLOG -> ss_cpplog
  • I_CPPPREFS -> ss_cppprefs
  • I_DIAMETERAPI -> ss_dia
  • I_FICM -> ss_ficm
  • I_HASLIB -> ss_pro
  • I_PM9 -> ss_pm9
  • I_PMGMTCAPI -> ss_libpmgmt
  • I_PMGMTEVENT -> ss_pmgmt
  • I_SCLI -> ss_scli
