The regression suites we use in FP5-Cat are FlexiTAF jobplans and use various metadata of the automated testcases (see [1]).
The jobplans are part of the testcase repository: http://sugar.fp.nsn-rdnet.net/cgi-bin/viewvc.cgi/rel_5.cat/FunctionalTesting/common/jobplans/

[edit] BI and BI related jobplans

  • jobplan1_BI.xml: Jobplan which selects only testcase that are executable on the target under test and have property "ProductIntegrationTestcase" set to "true" and property "AutomationLevel" set to "fullyAutomated". Testcases with set property "OpenBug" are excluded. Testcases have to be in domain "rel_5.cat/FunctionalTesting".
  • jobplan_all_PI.xml: Jobplan which selects all testcases from domain "rel_5.cat/FunctionalTesting" which have property "ProductIntegrationTestcase" set to "true" and property "AutomationLevel" set to "fullyAutomated".

[edit] Regression suite execution jobplans

  • jobplan2_regression_suite.xml: Jobplan which selects only testcase that are executable on the target under test and have property "AutomationLevel" set to "fullyAutomated". Testcases with set property "OpenBug" are excluded. Testcases have to be in domain "rel_5.cat/FunctionalTesting".
  • jobplan3_reexecute.xml: see common-use jobplans
  • jobplan3_reexecute_all.xml: see common-use jobplans
  • jobplan4_with_openbug.xml: Jobplan which selects all testcases executable on the target under test which are "under investigation" (i.e., have property "OpenBug" set). The testcases need to be in domain "rel_5.cat/FunctionalTesting" and have property "AutomationLevel" set to "fullyAutomated".
  • jobplan4_with_openpronto.xml: Jobplan which selects all testcases executable on the target under test which are known to currently detect a pronto (i.e., have property "OpenPront" set). The testcases need to be in domain "rel_5.cat/FunctionalTesting" and have property "AutomationLevel" set to "fullyAutomated".
  • jobplan5_nightly.xml: Jobplan that selects all testcases which are executable on the target under test and can be finished within the time period denoted by the afcli parameter -D job-duration="XX uu" (uu: minutes/hours/days/months/years). Only testcases from domain "rel_5.cat/FunctionalTesting" with property "AutomationLevel" set to "fullyAutomated" are selected. Testcases with property "OpenBug" are not selected.
  • jobplan6_domain.xml: see common-use jobplans
  • jobplan7_health_poor.xml: Jobplan that selects only testcases executable on the target under test which have reliability/health status set to "poor". Only testcases from domain "rel_5.cat/FunctionalTesting" with property "AutomationLevel" set to "fullyAutomated" are selected. Testcases with set property "OpenBug" are not selected.
    The health status of a testcase can be seen like this: afcli show testcase -r 5.cat FunctionalTesting/<DOMAIN>/<SUBDOMAIN> <TESTCASE>.

[edit] Common-use jobplans

  • jobplan6_domain.xml: Jobplan that selects all testcases from specific domain (denoted by afcli parameter -D domain-name=XXXX) which are executable on the target under test. Only testcases with property "AutomationLevel" set to "fullyAutomated" are selected.
  • jobplan3_reexecute.xml: Jobplan that selects only testcsaes which were failing in a previous job (denoted by afcli parameter -D job-result-id=XXXXX) and excludes all testcases with set property "OpenPronto" (those are expected to fail anyway because of the Pronto).
  • jobplan3_reexecute_all.xml: Jobplan that selects only testcsaes which were failing in a previous job (denoted by afcli parameter -D job-result-id=XXXXX)
