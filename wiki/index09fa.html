Hardware configuration: 

- ACPI2-A card in slot 7
 - 4 port ethernet card in upper AMC slot
 - HDSAM (Radisys Harddisk) in lower AMC slot

- ACPI2-A card in slot 10
 - 4 port ethernet card in upper AMC slot
 - HDSAM (Radisys Harddisk) in lower AMC slot

- AHUB2-A in slot 8 
- NO AHUB2-A in slot 9


You need console connection to ACPI2-A cards (38400n8)


NOTE These instructions expects that your AHUB2-A blade is configured NOT to use any VLANs (the default configuration in AHUB2-A uses VLAN 99). There are HW configuration instructions in your FEWS (/opt/nokiasiemens/SS_FEWS/doc/QUICKGUIDE_ATCA.txt). but they don't work. good luck.

NOTE2 Make sure you disable the "OS load timeout" in your ACPI2-A cards BIOS before starting the commissioning. The default timeout is "150" (seconds), change it to "0". Also change the "action" to "none" (it is by default "hard reset" and we don't want that to happen).

Its OK to leave "BIOS POST Timeout" enabled.

7. Connect the comissioning cable from the FEWS to AHUB2-A blade in slot 8 to the interface "BI P1" (the 4th interface counting from the bottom of the AHUB blade)

- Make sure the green link led next to interface is lit when you connect the cable. If not, the problem could be that your FEWS has 100M interface and the AHUB interface is 1G. It is up to you to solve that problem I.e buy 1Gig interface card to FEWS or use 1Gig/100M capable switch between the FEWS and AHUB2..

[edit] Starting commissioning

NOTE2 Make sure you disable the "OS load timeout" in your ACPI2-A cards BIOS before starting the commissioning. The default timeout is "150" (seconds), change it to "0". Also change the "action" to "none" (it is by default "hard reset" and we don't want that to happen).

You can access ACPI2-A BIOS by pressing F2 after reset of the ACPI2-A blade. (be quick to press it when you see any prints after ACPI2-A blade reset). Note that the blade won't enter the BIOS immediately; it will first load LSI SCSI BIOS and all the network card PXE boot stuff before dropping you to BIOS. However if you see the blade trying to boot from network or hard disk you were too slow with pressing F2..


8. Make sure the CLA-1 (the ACPI2-A card in slot 10) is not trying to boot (hotswap it out if you don't know other means to ensure this)

9.(re)start your CLA-0 node (the ACPI2-A card in slot 7) If you have setup the ACPI2 correctly to boot from network and the connection to FEWS is OK you will get the normal commsioning boot prompt; just type "kick" and hit enter to continue

The commissioning will start and continue for a while. When you see prints:

Then Run postconfig: 

[root@CLA-0(ATCA4_R2S1) /root]
# postConfig.py


2. Enable the root SSH access 

[root@CLA-0(ATCA4_R2S1) /root]
# vi /etc/ssh/sshd_external_config

Edit line: 

PermitRootLogin without-password

to: 

PermitRootLogin yes

Save changes and close the file.

Copy the conf to the other CLA ( in this case I copy it to CLA-1): 

[root@CLA-0(ATCA4_R2S1) /root]
# scp /etc/ssh/sshd_external_config CLA-1:/etc/ssh/sshd_external_config

And restart SSHd 

[root@CLA-0(ATCA4_R2S1) /root]
# fshascli -rn /CLA-0/FSDirectoryServer/SSHDaemon

3. All Done. Hopefully something works. you should be ablto SSH to your cluster with Directory IP. In my case i could do from "flegrp12":
