Commissioning steps (mixture of Sami's demo, the QUICKGUIDE and own "instructions" for file storage)

  • QUICKGUIDE for commissioning OcteonRM

Prepare Commissioning and Commissioning

  • copy is image from LinSEE (/workflexman/CI/iso/R_FSPR5CAT_1.<your_build_selection>.iso) to FEWS:/opt/nokiasiemens/fews/builds/
  • copy rpm image from LinSEE (/workflexman/CI/rpm/R_FSPR5CAT_1.<your_build_selection>.rpm) to FEWS:/tmp/
  • extract SS_FEWS from build's rpm: 
 FEWS: rpm2cpio /tmp/R_FSPR5CAT_1.<your_selection>.rpm | cpio -id "*SS_FEWS*"
  • remove old SS_FEWS (if needed): 
 FEWS: rpm -e SS_FEWS
  • install new SS_FEWS: 
 FEWS: rpm -ihv --nodeps --ignorearch /tmp/var/mnt/local/sysimg/flexiserver/opt/Nokia/var/swmgmt/downloaded_rpms/R_FSPR5CAT_1.<your_build_selection>/SS_FEWS-.<your_build_selection>*.rpm
  • patch delivery: 
 FEWS: /opt/nokiasiemens/SS_FEWS/bin/fsetupcli --delivery --patch /opt/nokiasiemens/fews/builds/R_FSPR5CAT_1.<your_build_selection>.iso
  • create/update fsetup config file /opt/nokiasiemens/fews/etc/<your_cluster_name>/fsetup.conf
  • create/update ipscript file /opt/nokiasiemens/fews/etc/<your_cluster_name>/ipscript.sh
  • create commissioning session: 
 FEWS: /opt/nokiasiemens/SS_FEWS/bin/fsetupcli --session --create <your_session_name> /opt/nokiasiemens/fews/etc/<your_cluster_name>/fsetup.conf
  • start the commissioning session: 
 FEWS: /opt/nokiasiemens/SS_FEWS/bin/fsetupcli --session --start <your_session_name>
  • check CPU1 and CPU2 console uBoot settings (see QUICKGUIDE section 2.2)
  • start netboot installation
  • reboot cluster (reboot -f)
  • during booting press "space" to enter uBoot prompt
  • follow instructions in QUICKGUIDE section 5
  • wait until commissioning finishes, then stop session, and reboot cluster
  • run postConfig 
 /opt/nokiasiemens/bin/postConfig.py
  • enable ssh root login 
 /etc/ssh/sshd_external_config : PermitRootLogin => yes, usePAM => no
  • restart sshd (hascli -r /CLA-0/FSDirectoryServer/SSHDaemon)
  • check that ssh login as root from outside works 
 flexiaf*: ssh root@<CLUSTER_DIRECTORY_IP>

Prepare for automated testing with FlexiTAF

  • unlock TargetInfo RG and RUs 
 hascli -u /TargetInfo /CLA-0/FSTargetInfoServer
  • check that target information gets updated in afcli after some time (can be up to 20 minutes) (or add target if needed with "afcli add target <CLUSTER> tcp://<TARGET_INFO_IP>:443") 
 flexiaf*: afcli show target <CLUSTER>
  • change the prompt on the cluster by commenting the line with PS1 in the file .profile (this is for one current Pronto) 
 cluster: vi /root/.profile

Run testcases with FlexiTAF

  • try run some automated testcases (e.g., FT_OS_039) 
 afcli -r 5.cat -t <CLUSTER>  -u root -p root exec FT_OS_039
