Connecting the target

  • tc: 
#!/bin/bash
echo "Hello World"
  • tc_analyze: 
#!/bin/bash
if grep "Hello World" $TC_OUT
then
   exit TCE_EXIT_PASS;
fi
exit TCE_EXIT_ERROR;
  • Test case logic is placed in tc script. Parts of tc script are executed in the target, in local host, or in other hosts depending on the macros the tc scripts contains.
  • First part of tc script, before the first %-macro, is always executed in the target (but the first part can be empty).
  • Analysis logic is placed in tc_analyze that will be excecuted in the local host after tc has finished.
  • The exit code of tc_analyze is the result of the testcase. If tc_analyze is missing the exit code of tc is the result.

Opening connection many times

  • tc: 
#!/bin/bash
echo "Hello World"

%NODE none
#!/bin/bash
echo "Hello Again"

%NODE none
#!/bin/bash
echo "Hello Third Time"
  • tc_analyze: 
if grep "Hello World" $TC_OUT
then
  if grep "Hello Third Time" $TC_OUT
    exit TCE_EXIT_PASS;
  fi
fi
exit TCE_EXIT_ERROR;
  •  %-macros split the tc script to smaller scripts that will be executed separately.
  • "%NODE none" opens connection to the primary target.

Opening connections to different nodes

  • tc: 
#!/bin/bash
uname -n

%NODE AS-1
#!/bin/bash
uname -n
  • tc_analyze: 
if grep "AS-1" $TC_OUT
then
  exit TCE_EXIT_PASS;
fi
exit TCE_EXIT_ERROR;
  • "%NODE <name>" opens connection to a named node

Executing the same script in many nodes

  • tc: 
%NODE AS-1, AS-2, AS-3, AS-4, AS-5
#!/bin/bash
uname -n
  • "%NODE <name>" also accepts list as an argument

Executing scripts simultaneously

  • tc: 
%BACKGROUND 1 NODE IPD-0-A
#!/bin/bash
tethereal -i any -R icmp

%NODE none
#!/bin/bash
ping -c 10 $PING_TARGET

%BACKGROUND 1 KILL
  • tc_analyze: 
if grep "ICMP Echo" $TC_OUT_BG_1
then
  exit TCE_EXIT_PASS;
fi
exit TCE_EXIT_ERROR;
  • "%BACKGROUND <id> NODE <name> starts a script in the background and immediately jumps to the the next macro.
  • "%BACKGROUND <id> KILL" is used to kill background process.
  • "%BACKGROUND <id> WAIT" can be used to wait for background process. The background process must not have been killed before TCExec starts to wait for it.

Executing instances of the same script in parallel

  • tc: 
%BACKGROUND 1,2,3,4,5 NODE AS-1,AS-2,AS-3,AS-4,AS-5
#!/bin/bash
uname -n
  • "%BACKGROUND <id list> NODE <name list>" also works
  • A testcase won't finish until all background tasks have finished, or they have been killed.

Rebooting the target

  • tc: 
#!/bin/bash
uptime
reboot

%SLEEP 30

%NODE node
#!/bin/bash
uptime
  • If the target becomes unavailable in the middle of testcase execution the next %NODE macro retries to open the connection until it succeeds.
  • It is sometimes necessary to add %SLEEP to make sure that target goes down before the next connection is opened.

Transferring data between %-macros

  • tc: 
%NODE none
#!/bin/bash
echo "export MY_MESSAGE=\"Hello World\" >> $TCE_USER_ENV

%UPDATE_USER_ENV

%NODE none
#!/bin/bash
echo $MY_MESSAGE

%LOCAL
#!/bin/bash
echo $MY_MESSAGE
  • $TCE_USER_ENV is a special script that is sourced before scripts are executed.
  •  %UPDATE_USER_ENV macro copies the user environment script from the target to controlling host so that it is available for following scripts.
  • As all macros %UPDATE_USER_ENV ends the preivious script, so it cannot appear in the middle of a bash script. New connection (%NODE, %LOCAL, %HOST) has to be opened after calling %UPDATE_USER_ENV
