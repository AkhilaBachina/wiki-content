Currently the Bobcat build needs to be modified before it can be comissioned to ATCA system. Below are the instructions how.

NOTE These instructions expects that your AHUB2-A blades are configured NOT to use any VLANs (the default configuration in AHUB2-A uses VLAN 99). There are HW configuration instructions in your FEWS (/opt/nokiasiemens/SS_FEWS/doc/QUICKGUIDE_ATCA.txt). but they don't work. good luck.

NOTE2 Make sure you disable the "OS load timeout" in your ACPI2-A cards BIOS before starting the commissioning. The default timeout is "150" (seconds), change it to "0". Also change the "action" to "none" (it is by default "hard reset" and we don't want that to happen).

Its OK to leave "BIOS POST Timeout" enabled.

Preparing the build for commssioning in FEWS

In your FEWS:

1. Get the build to your FEWS: 

cp /build/CI/iso/R_FSPR5CAT_1.8.1.0.WR.32.0808080920.4_GenericWRx86.iso /opt/nokiasiemens/fews/build/

or you can fetch the build from LINSEE servers with FTP/SCP if you don't have build directory (/build/CI) mounted in your FEWS.

2. Installing the correct SS_FEWS package on FEWS 2.1 Remove the SS_FEWS package and install the one from build 

rpm -q SS_FEWS
rpm -e SS_FEWS
cd /tmp/
rpm2cpio /build/CI/rpm/R_FSPR5CAT_1.8.1.0.WR.32.0808080920.4_GenericWRx86-base-1.rpm | cpio -id "*SS_FEWS*"

Again You can get the base rpm from LINSEE server with FTP/SCP if you don't have the build directory mounted in your FEWS.

2.2 Install the SS_FEWS package 

rpm -ivh --nodeps --force /tmp/var/mnt/local/sysimg/flexiserver/opt/Nokia/var/swmgmt/downloaded_rpms/R_FSPR5CAT_1.8.1.0.WR.32.0808080920.4_GenericWRx86/SS_FEWS-1.12-R_FSPR5CAT_1.7.1.2.WR.i386.rpm 
rpm -q SS_FEWS

3. Patching the build 

/opt/nokiasiemens/SS_FEWS/bin/fsetupcli --delivery --patch /opt/nokiasiemens/fews/build/R_FSPR5CAT_1.8.1.0.WR.32.0808080920.4_GenericWRx86.iso

4. Editing the build image 

mkdir /mnt/image
mount /opt/nokiasiemens/fews/deliveries/R_FSPR5CAT_1.8.1.0.WR.32.0808080920.4_GenericWRx86/R_FSPR5CAT_1.8.1.0.WR.32.0808080920.4_GenericWRx86.img /mnt/image/ -o loop
vi /mnt/image/opt/nokiasiemens/SS_IPMgmt/script/commissioning-ip-script.sh

Change in the (almost) beginning of the file line: 

# These environment variables can be used to control this script:
# - COMM_IPMGMT_INTERNAL_IFACE

to 

# These environment variables can be used to control this script:
COMM_IPMGMT_INTERNAL_IFACE=eth0

And litlle bit lower change line: 

# - COMM_IPMGMT_FI_IFACE

to 

COMM_IPMGMT_FI_IFACE=eth2

Save changes and close that file.

5. Unmount the image 

cd /opt/nokiasiemens/fews/
umount /mnt/image/

Ok image is now ready to be commissioned

7. Edit the ipscript (if needed) Here is example (/opt/nokiasiemens/fews/ipscript_atca.sh) 

#!/bin/bash -x
echo ATCA4R2S1-IPSCRIPT-START

fsipnet ethiface add eth4 addphase nwmgrstart delphase never node CLA-0 owner /CLA-0 hw JOKULINKKI
fsipnet ethiface add eth4 addphase nwmgrstart delphase never node CLA-1 owner /CLA-1 hw JOKULINKKI
fsipaddress dedicated add 10.201.101.131/24 owner /CLA-0 iface eth4
fsipaddress dedicated add 10.201.101.132/24 owner /CLA-1 iface eth4
fsiproute add 0.0.0.0/0 owner /CLA-0 nexthop via 10.201.101.1 iface eth4
fsiproute add 0.0.0.0/0 owner /CLA-1 nexthop via 10.201.101.1 iface eth4 

fsipaddress dedicated add 10.201.101.130/24 iface eth4 owner /Directory
fsipaddress dedicated add 10.201.101.133/24 iface eth4 owner /TargetInfo
fsipaddress dedicated add 10.201.101.134/24 iface eth4 owner /ClusterNTP
fsipaddress dedicated add 10.201.101.135/24 iface eth4 owner /ClusterDNS 

fsipnet service add ClusterNTP forwarder 10.201.101.1
fsipnet service add ClusterDNS forwarder 10.102.12.68 forwarder 10.102.12.69

echo ATCA4R2S1-IPSCRIPT-STOP
exit 0

9. Connect the comissioning cable from the FEWS to AHUB2-A blade in slot 8 to the interface "BI P1" (the 4th interface counting from the bottom of the AHUB blade)

- Make sure the green link led next to interface is lit when you connect the cable. If not, the problem could be that your FEWS has 100M interface and the AHUB interface is 1G. It is up to you to solve that problem I.e buy 1Gig interface card to FEWS or use 1Gig/100M capable switch between the FEWS and AHUB2..

Starting commissioning

NOTE2 Make sure you disable the "OS load timeout" in your ACPI2-A cards BIOS before starting the commissioning. The default timeout is "150" (seconds), change it to "0". Also change the "action" to "none" (it is by default "hard reset" and we don't want that to happen).

You can access ACPI2-A BIOS by pressing F2 after reset of the ACPI2-A blade. (be quick to press it when you see any prints after ACPI2-A blade reset). Note that the blade won't enter the BIOS immediately; it will first load LSI SCSI BIOS and all the network card PXE boot stuff before dropping you to BIOS. However if you see the blade trying to boot from network or hard disk you were too slow with pressing F2..

10. (re)start your CLA-0 node (the ACPI2-A card in slot7) If you have setup the ACPI2 correctly to boot from network and the connection to FEWS is OK you will get the normal commsioning boot prompt; just type "kick" and hit enter to continue
