24.09.2008

UPDATE


CAT MIPS 

Last Working Build: R_FSPR5CAT_1.10.WR.mips.0809230700.2076

- Node fails to boot up since mkboot command is not executed. Aliaksei K. is looking into the problem. No pronto has been raised yet.

- Occassionally, slapd does not get killed while commissioning. Madhu is looking into the problem. Some additional debug has been enabled to investigate the problem.

- Some LISA RGs are in NOK state in build R_FSPR5CAT_1.10.WR.mips.0809230700.2076

- Occassionally, the cluster reboots. Tomi Juvonen has provided the fix but needs to be tested.

- *** NMI Watchdog interrupt. Chip soft reset *** problem has reappeared. Jukka V. is investigating this issue.


CAT Virtual Environment 

Last working build: R_FSPR5CAT_1.10.WR.32.0809081003.2019

- AS nodes fail to bootup due to kernel map problem. PR 26791ESPE01: Booting up of AS nodes fail due to kernel image map problem (truncation of RamDisk image name ) has been raised for Foundation team.

- Directory takes some time to come up. Vipul needs to discuss this with TLDAP team.

- With R_FSPR5CAT_1.10.WR.32.0809221900.2076 build onward there have been problem with directory. The directory comes up and then goes down due to some major RU failure and the comes up again after sometime and then stays enabled. Issue needs to be investigated.


CAT ATCA 

Last working build: R_FSPR5CAT_1.10.WR.32.0809221100.2070

- Build R_FSPR5CAT_1.10.WR.32.0809221700.2075 doesn't work, some interfaces are missing. Aliaksei K. is investigating the issue.

- RoutingDaemon doesn't work because deployment defines fsipIPAddrType as Redundant. Tomi J. has promised a fix at the end of the week.

- Default route doesn't get added automatically. It can be added manually. Otilia B. is working on this and fix is expected at the end of the week.

- AS nodes does not come up (implementation missing). Work ongoing.

23.09.2008

UPDATE


CAT MIPS 

Last Working Build: R_FSPR5CAT_1.10.WR.mips.0809230700.2076

- NOTE: Mostly same as Friday, fixes ongoing.

- Occassionally, slapd does not get killed while commissioning. Madhu is looking into the problem. Some additional debug has been enabled to investigate the problem.

- Tab does not work with hascli.

- Some LISA RGs are in NOK state in build R_FSPR5CAT_1.10.WR.mips.0809230700.2076

- Occassionally, the cluster reboots. Foundation team has been informed about it. Tomi Juvonen needs to provide fix for this.

- *** NMI Watchdog interrupt. Chip soft reset *** problem has reappeared. Jukka V. is investigating this issue.


CAT Virtual Environment 

Last working build: R_FSPR5CAT_1.10.WR.32.0809081003.2019 (same as Friday)

- AS nodes fail to bootup due to kernel map problem. PR 26791ESPE01: Booting up of AS nodes fail due to kernel image map problem (truncation of RamDisk image name ) has been raised for Foundation team.

- Directory takes some time to come up. Vipul needs to discuss this with TLDAP team.

- With R_FSPR5CAT_1.10.WR.32.0809221900.2076 build onward there have been problem with directory. The directory comes up and then goes down due to some major RU failure and the comes up again after sometime and then stays enabled. Issue needs to be investigated.

CAT ATCA 

Last working build: R_FSPR5CAT_1.10.WR.32.0809221100.2070

- Build R_FSPR5CAT_1.10.WR.32.0809221700.2075 doesn't work, some interfaces are missing. Aliaksei K. is investigating the issue.

- RoutingDaemon doesn't work because deployment defines fsipIPAddrType as Redundant. Tomi J. has promised a fix at the end of the week.

- There was a problem with CLA-nodes booting from each other. This has been fix but introduced an another problem. Now CLA-nodes need to be in slots that are defined in deployment (which is good), but deployment defines wrong slots. Work-a-round is to temporarily locate CLA-0 in slot 1 and CLA-1 in slot 2. Tomi J. will provide a fix for this at the end of the week.

- Default route doesn't get added automatically. It can be added manually. Otilia B. is working on this and fix is expected at the end of the week.

- More services will be added.

- AS nodes does not come up (implementation missing). Work ongoing.
