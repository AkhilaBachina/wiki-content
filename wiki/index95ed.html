How to read the status reports

  • the following graphs represent the current status of the FP4 Charm regression suite for MCP18 HW which is set up by TATF
  • the graphs show the progress of TATF (i.e., at the moment they are not necessarily an indicator for the automation level)
  • the process how TATF selects testcases to be added to the regression suite is described here
  • details of the 3 columns:
    • planned testcases: all planned testcases as defined in the testplan and as already processed by TATF (i.e., there might be more planned testcases than shown here)
    • automated testcases: all testcases where a fully automated testcase implementation is available in the testcase repository and as already processed by TATF (i.e., there might be more fully automated testcases than indicated here)
    • testcases in regression suite: testcases that have already been identified by TATF to be part of the respective regression suite (i.e., there might be more testcases in the regression suite later on)
  • executions of the regression suite can be found here
  • FAQ:
    • Why are there more automated testcases than testcases in the regression suite?
      • some testcases need too long to be executed in a regression suite, TATF considers only testcases which need maximum 30-40 minutes
      • not all testcases are yet in the shape to be added to a regression suite (more info here)
      • setting up the regression suite is work in progress and identifying a testcase as automated and adding it to the regression suite is not the same
    • Why do the statistics in FlexiTAF WebUI show more automated testcases for one area?
      • only fully automated testcases are considered for the regression suite, some areas have some testcases which need manual evaluation
      • setting up the regression suite is work in progress, so TATF may not yet have finished processing this area or need to re-process it after new testcases have been added to the repository
      • some graphs considers only mandatory or optional testcases but the FlexiTAF WebUI does not distinguish them
      • FlexiTAF WebUI considers all testcases that are in the repository including those that might be "not possible" on MCP18 HW
