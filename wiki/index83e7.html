[edit] FP5 Cat BI FAQ

[edit] Q1. How to enable remote connection to a virtual node?

  1. During commissioning, both FEWS and Commissioning node should be in “host-only” network so that the node can boot from FEWS.
  2. Assign IP Address to external network e.g. NOKLAB in the IP script.
  3. After commissioning, stop the session in FEWS and shut down the node.
  4. Change the node network interface to “bridged”.
  5. Boot up the node

Note: Check with Lab support for a free IP Address

[edit] Q2. How to enable remote ssh root login after commissioning?

1. Edit external_sshd_config config file (/var/mnt/local/sysimg/flexiserver/opt/Nokia_BP/etc/ssh/external_sshd_config) in active directory node (e.g. CLA-0):

Change: 

PermitRootLogin no

To: 

PermitRootLogin yes

2. Restart ssh daemon: 

# hascli -r /CLA-0/FSDirectoryServer/SSHDaemon

Replace CLA-0 with CLA-1 if it's active directory node.

You can check which one is active for example using INTERNAL ssh to login to active directory node: 

# ssh directory

[edit] Q3. Why I get tftp time-out when I try to boot up virtual node from virtual FEWS for commissioning?

A: In VMWare environment node location is decided by virtual machine's MAC address. If you have another virtual node running with same MAC address in "host-only" network, tftp cannot resolve arp query. Shut down another node during commissioning or remove it from the "host-only" network.

[edit] Q4. What is the fastest way to see FlexiPlatform source code?

A: Go to the Flexiplatform SVN pages: 

http://halti.ntc.nokia.com/FlexiServer_SCM/CommonSCM/product/FlexiServerPlatform/ssmap.htm

Note! All subsystems cannot be seen in SVN.

Choose the correct subsystem(eg. SS_IPMgmt) and follow path: 

/tags/SS_IPMGMT_6.8/SS_IPMgmt/script/01-SS_IPMgmt_translate.sh

[edit] Q5. What is the fastest way to try a bug fix for a script in FlexiPlatform?

1.mount the latest build/image on FEWS 

#mkdir /mnt/buildx
#mount /opt/nokiasiemens/fews/deliveries/R_FSPR5CAT_1.5.WR.mips.0804101235.1539_GenericWRmips/R_FSPR5CAT_1.5.WR.mips.0804101235.1539_GenericWRmips.img /mnt/buildx -o loop

2.Find the location of changed file if you don't know. 

# find /mnt/buildx -name "*SS_IPMgmt*"

/mnt/buildx/opt/nokiasiemens/fews/bin/01-SS_IPMgmt_translate.sh


3.copy the latest changed file supplied by developer to FEWS server and then to the mounted image. 

#cp /root/01-SS_IPMgmt_translate.sh /mnt/buildx/opt/nokiasiemens/SS_FSetup/bin/01-SS_IPMgmt_translate.sh

4.unmount & start the commissioning again 

#umount /mnt/buildx

[edit] Q7. How to update OcteonRM bootloader SW?

1. Copy bootimage(fwa5510a1_vx.x.x.bin) to FEWS /tftpboot/ directory

2. load image to flash: 

0H#FWA-5510A1$ fatload ide 0 0x400000 fwa5510a1_vx.x.x.bin

3. Update flash: 

0H#FWA-5510A1$ run bootloadet_flash_update
0H#FWA-5510A1$ reset

4. Check version: 

0H#FWA-5510A1$ version

U-Boot 1.1.1 vx.xx

[edit] Q8. New version SS_FEWS was published. Where do I get it and how to I install it?

When new version of SS_FEWSis published, it needs to be installed to FEWS. Unfortunatelly there is no separate SS_FEWS rpm available yet, so it needs to be extracted from the delivery rpm first. Delivery rpm is not used for anything else any more.

1. Create a temporary directory in LinSEE and cd to it. 

# mkdir /tmp/mydirectory
# cd /tmp/mydirectory
#

2. Delivery rpms are located in /build/CI/rpm/. Extract SS_FEWS from delivery rpm. It will be extracted to var/mnt/local/sysimg/flexiserver/opt/Nokia/var/swmgmt/downloaded_rpms/<name of the delivery>/ under current directory 

# rpm2cpio /build/CI/rpm/R_FSPR5CAT_1.7.WR.mips.0806031354.1711_GenericWRmips-base-1.rpm | cpio -id "*SS_FEWS*"
# ls var/mnt/local/sysimg/flexiserver/opt/Nokia/var/swmgmt/downloaded_rpms/R_FSPR5CAT_1.7.WR.mips.0806031354.1711_GenericWRmips/SS_FEWS-1.9-R_FSPR5CAT_1.7.WR.mips.rpm
SS_FEWS-1.9-R_FSPR5CAT_1.7.WR.mips.rpm
#

3. Copy SS_FEWS to FEWS 

# scp var/mnt/local/sysimg/flexiserver/opt/Nokia/var/swmgmt/downloaded_rpms/R_FSPR5CAT_1.7.WR.mips.0806031354.1711_GenericWRmips/SS_FEWS-1.9-R_FSPR5CAT_1.7.WR.mips.rpm root@rancid2.fp.nsn-rdnet.net
#

4. SSH to FEWS and remove previous version of SS_FEWS from FEWS if any. You can ignore the warning about missing file/directory. 

# ssh root@rancid2.fp.nsn-rdnet.net
# rpm -q SS_FEWS
SS_FEWS-1.9-R_FSPR5CAT_1.7
# rpm -e SS_FEWS
error: failed to stat /opt/nokiasiemens/fews/deliveries/patch.M32014/mnt: No such file or directory
# rpm -q SS_FEWS
package SS_FEWS is not installed
#

5. Install new SS_FEWS to FEWS. You can ignore the warning about missing file/directory 

# rpm --nodeps --ignorearch -ivh SS_FEWS-1.9-R_FSPR5CAT_1.7.WR.mips.rpm
error: failed to stat /opt/nokiasiemens/fews/deliveries/patch.M32014/mnt: No such file or directory
Preparing...                ########################################### [100%]
   1:SS_FEWS                ########################################### [100%]
#
