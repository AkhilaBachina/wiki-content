21.05.2008

UPDATE

BUILD: R_FSPR5CAT_1.7.WR.mips.0805211531.1664, with LISA deployment

  • OS version was upgraded with fix for nfs (kernel boot parameter, done by Suvendu to SS_FSetup) to make it work. Commissioning now fails in creating the session in FEWS. Anup M. is investigating the issue.

FROM LOG:

[root@rancid1 sessions]#
  • Pula freeze is still in effect.
  • LISA services are not coming up. Baha M. has been investigating the issue and found several problems:
    • There are several bugs in LISA CAM process.
    • Deployment is containing only 1 RU under LISADomain RG when there should be 3 RUs. Tomi J. needs to fix this.
  • OS still needs to be updated later to include proper hwclock implementation and rdate command.
  • SNMPMediator is still not working, there has been an open pronto a while for it.
  • Pending pulas are now taken into the build, one team at a time. Foundation and Deployment and Storages teams have pulad already.
  • Tomi J. informed about a huge deployment pula later this week. This can cause possible breakage of the build: 
Hi,

There will be many things pulad in this week in the deployment area:
- ATCA support (create_1chassis_nsnatca_bp.sh maybe other new deployments also)
- New SAN structure (requires also HAS and SS_Sat changes also with Octeon RM, not just ATCA)
- Transactional LDAP support for deployment (transaction ID is passed to every utility also as "None" during  commissioning)
- fshaAbsolutePath -> fshaProcessCommand in FSHAProcess object

Changes are tight together in a way that they should be pulad in all at once. This might cause build to be broken some time case we do not have a method to make own test build separately.

-Tomi
