Introduction

Using mock testing you can easily test interactions between the objects in your application. If your application requires complex external dependencies, you can simulate the availability of these resources by creating mock objects for them and then inject these mock dependencies to your application in your test fixture setup.

Useful dependencies to mock are for example: processes, remote service, database data access objects, etc.

Mock objects can be also programmed to simulate error cases.

You can use mock objects to also verify behavior of objects by progamming constraints for you mock object how it should be called during the tests. Mock frameworks can then verify the constraints and determine was the test passed or failed.

Mock frameworks

JMock

With Java you can utilize JMock [1]. It can be easily integrated with Maven to JUnit tests [2]. Examples can be found from cookbook [3]. It uses internally Hamcrest for defining matching rules [4].

JMock is used in FlexiTAF and an example can be found from there [5]. In that example we are mocking target-discoverer, which is required dependency of target-repository, but to make testing easier we don't want to setup full SOAP client/server framework.
