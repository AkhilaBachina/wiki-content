General rules

  • Stick to one coding convention in single project. If you feel that it should be changed, then change it to whole project.
  • Use some editor which can do reasonable indentation automatically
    • Configure your editor to indent using spaces instead of tab characters. See Editor configuration.
  • If you use object-oriented languages, write also object-oriented code.

C++

Naming

  • Class names start with capital letter and doesn't contain any underscores '_' 
   class Person{};
  • Method names start with non-capital letter and doesn't contain any underscores '_' 
   void getName();
  • It is good practice to distinguish different type of variables 
   class Person
   {
       int m_age;
       Person* m_pFriend;
       void setAge(int age);
   };
   int g_personCounter;
  • Namespaces should start with non-capital letter and doesn't contain any underscores '_' 
   namespace testdia
   {
   }
  • Template argument names should start with capital letter 
   template<typename T, typename OtherType>

Coding

  • Don't use macros for coding purposes
  • Use STL
  • Don't use throw specifications
