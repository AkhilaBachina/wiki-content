[edit] 06.06.2008

UPDATE

BUILD: R_FSPR5CAT_1.7.WR.mips.0806052100.1725

  • Commissioning works with bp and bp lisa deployments with normal CI build.
  • Full deployment with some fixes from Tomi J. were tried but there are more problems. Commissioning succeeds but node is in reboot loop afterwards. It appears LDAP database in target is missing user account data. Jan C. is investigating the issue. More information is expected on Monday.
  • There are some issues with BCQ iso creation, Jussi M. is investigating.
  • There will be info session and FP Virtual Operations recreation day Monday, most of the people are going so this will affect swd, testing and BI.


  • Pasi J. noted about following mail discussion: 
Subsystems having conflicts with OS files (this concerns now only 
SS_AAA, SS_BPConf, SS_BIT) should follow this arrangement:
- subsystem installs the conflicting files to to different location than OS
- e.g. /etc/nsswitch.conf is installed to  /opt/nokiasiemens/SS_AAA/etc/nsswitch.conf
- subsystem has finalization script, which overwrites the default  files
- e.g. cp -f /opt/nokiasiemens/SS_AAA/etc/nsswitch.conf /etc/nsswitch.conf
Appropriate access permissions should be also set.
  • Fixes for this have been published by Tuomo T. (SS_BIT and also SS_IPMGMT was needed) and Petri H. (SS_AAA). Fix for SS_BPConf is still missing.


  • There is a minor issue with SS_FEWS, if fsetup.conf debug option is set to true there will be mount to delivery and link to it left in SS_FEWS directory which causes error messages while installing and uninstalling the package. This does not affect the actual installation or uninstallation. Tomi A. is already fixing this.

[edit] 05.06.2008

UPDATE

BUILD: Various patched builds

  • NFS problem has been tracked to kernel driver, with old kernel Aaro was able to reproduce the problem and with new test kernel commissioning succeeded. New OS will be available next week, but it might be a good idea to delay taking it into use after new official build, because it might introduce other problems.
  • Execution order of the commissioning scripts has been changed in Cat compared to Charm. Now they are executed in alphabetical order, which caused RUIM-RMEX to fail; groups/users were tried to use before they were created. Work-a-round was to add two-digit number before the script names as suggested in RTE specification. There were also a problem with SS_BPConf which fails in commissioning. Eero K. provided a hack and patch for it. Now commissioning succeeds but fails to boot up with full lisa deployment.
  • Build works now with bp lisa deployment! This still needs to be tested with normal CI build.


  • Pasi J. noted about following mail discussion:

  • Fixes for this have been published by Tuomo T. (SS_BIT and also SS_IPMGMT was needed) and Petri H. (SS_AAA). Fix for SS_BPConf is still missing (there were some changes that broke the build).

This does not affect the actual installation or uninstallation. Tomi A. is already fixing this.
