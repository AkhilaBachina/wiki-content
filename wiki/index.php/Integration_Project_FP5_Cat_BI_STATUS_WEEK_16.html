UPDATE 2

  • Commissioning failed with similar errors. Issues under investigation.

UPDATE

BUILD: R_FSPR5CAT_1.5.WR.mips.0804161446.1560

  • Commissioning failed: 
DEBUG  Starting makepartion.py
DEBUG  rm -rf /tmp/fsPlatform-ldbm/*
DEBUG  cd /etc/opt/nokiasiemens/ldapfiles;/usr/sbin/slapd -h ldap://localhost:389 -f fsPlatformComm.conf &
ERROR  ldap startup failed:
ERR:make_partition failed

An earlier build (*.1556) failed also. Issue has been tracked to SS_MOMFSSECURITY_2.49 change, flexman downgraded it to previous version:
FlexiSCM ris R_FSPR5CAT_1.5: 'update' ann to system:SS_MOMFSSECURITY_2.48.
New build is under compilation.

  • It seems that GNU ld/dl-cache implementation is missing support for MIPS, and the linker is always rejecting any cache entries and uses the default system search paths instead. Aaro K. will make a support request to WR, but it will probably take few weeks to get it fixed. There is a work-a-round by adding paths directly. This is done by a fix from Pekka S. which has been pulaed but has not been tested yet.
  • Jukka V. announced new SS_Pro, SS_Sta and SS_SMR versions. These should fix the stack size setting & /rbin/ip reference problems.

ALL HAS COMPONENTS NEED TO BE RECOMPILED (SS_Sta, SS_Sup, SS_CFM, SS_Pro, SS_HUI) because there are interface changes, so these are not yet tested.

  • Gideon pulaed fix for path problems. This has not been tested yet, because the build was broken.
  • Second Octeon is also now in use.

TODO

  • Waiting for a new build from flexman and commission it to make sure all the fixes work and no other issues appear from changes that have been pulaed to the build. HAS needs to be rechecked for errors. New build should be available by tomorrow.
