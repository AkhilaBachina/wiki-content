  • ...


Subtask: Setting up the FP4 Charm regression suite

  • analyze which testcases are planned for FP4 Charm (and are supposed to be executed on which HW type) using the test plans in TestMan
  • check which of them are available as fully-automated testcases (i.e., testcases with manual checking of results or manual intermediate steps are not supposed to be fully automated!)
  • for setting up the FP4 Charm regression suite for IBM BC:
    • check which of them are currently passing on our test cluster Tornado and print no error message in the log files (e.g. checking for error messages like "No such file or directory")
    • select those which are passing and have a maximum duration from 30-40 minutes

Subtask: Executing the FP4 Charm regression suite

  • FP4 Charm regression suite for IBM BC is executed on IBM BC cluster "Tornado" (2-chassis cluster with IPD nodes)
  • executing the regression suite means automated commissioning of the latest release (and corrections) and executing the set of selected testcases; FlexiAutoCom script is used for these two tasks
  • execution frequency:
    • nightly whenever there is a new release build or correction
    • nightly whenever there are new testcases in the regression suite
    • (nightly whenever some of testcases have been changed)

FP4 Charm Regression Suite

The FP4 Charm Regression Suite is executed for new builds or each time when automated testcases have been added. Executing the regression suite means

  • doing automated commissioning with the latest build (configuration specifies if it is latest CI build or latest release build)
  • executing the set of selected testcases

Regression Suite testcase lists

The testcases that can be added to the regression suite are

  • automated testcases which can be executed with FlexiTAF (i.e, using the afcli command),
  • testcases which provide fully automated test evaluation,
  • testcases which are working reliable and leave the cluster in a condition that further testcases can be executed.
  • testcases with a usual duration of maximum 30-40 minutes

The list of testcases is currently defined in the following jobplan file. The syntax of this files is defined in the FlexiTAF User Guide.

IBM BladeCenter with IPD nodes Tornado /twafs/utilities/RegressionSuite/FP4_CHARM/fp4CHA_regression_suite_testcases.xml
NokiaBlade with UAN30 deployment Drought /twafs/utilities/RegressionSuite/FP4_CHARM/fp4CHA_regression_suite_NokiaBladeHW_testcases.xml

After adding testcases to the jobplan file, please test that the jobplan file can be correctly parsed using the following command: 

 afcli -t Tornado -f /twafs/utilities/RegressionSuite/FP4_CHARM/fp4CHA_regression_suite_testcases.xml --dump-execution jobplan 

 afcli -t Drought -f /twafs/utilities/RegressionSuite/FP4_CHARM/fp4CHA_regression_suite_NokiaBladeHW_testcases.xml --dump-execution jobplan

Regression Suite start commands

The regression suite (i.e. automated commissioning of latest CI build and then execution of the testcases) is started with the following command. It starts a script which is a wrapper for FlexiAutoCom (providing the same parameters as FlexiAutoCom) but has pre-defined value for the FlexiAutoCom configuration file to be used. Paramters that might want to be used are "-s", "-d", "-v" and "-vv". For their usage refer to the FlexiAutoCom UserGuide.

IBM BladeCenter with IPD nodes Tornado /twafs/utilities/RegressionSuite/FP4_CHARM/Tornado_comm_and_rsuite.sh
NokiaBlade with UAN30 deployment Drought /twafs/utilities/RegressionSuite/FP4_CHARM/Drought_comm_and_rsuite.sh

Before this command can be executed, an own commissioning environment has to be set up:

  • to allow ssh connection to esfleg* from flexiaf* without any password and
  • additionally enable ssh-key forwarding using a ssh-agent where all necessary keys have been added.

For instructions see the FlexiAutoCom Quickguide.

FEWS "ixion"

Required interface configuration on FEWS before commissioning:

  1. FEWS connection to grey zone: eth1
  2. FEWS commissioning interface: will be automatically added by FSetup to eth0.350 (IP address: 192.168.0.249/16)
  3. FEWS interface to connect to MMs: eth0 (no vlan!)
  4. Note: No IP address from the network 192.168.0.xxx/16 should be configured to interface eth0. If configured, remove automatic configuration from file /etc/sysconfig/network-scripts/ifcfg-eth0 on FEWS.

If the interface configuration has to be changed, follow these rules: FEWS interface configuration
